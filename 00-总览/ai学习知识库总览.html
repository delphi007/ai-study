如何使用本库

  1. 顺序阅读：建议从 Prompt Engineering 开始（先学会怎么说清楚），然后按 00 → 01 → … → 10 顺序阅读，前文是后文的基础
  2. 按需跳转：文章之间通过 [[双向链接]] 关联，可自由跳转探索
  3. 分层学习：先读”人机交互层”（11）学习如何表达，再读”扩展能力层”（01-03）建立基础，再读”协作执行层”（04/08/09），最后读”持久化与自动化层”（05-07）
  4. 实战优先：每篇文章包含 3+ 个软件开发和项目规划的真实示例
  5. 模板复用：新建专题时可使用 专题文章模板 保持一致性

学习路径建议

  1. 先读 Prompt Engineering，学会”如何跟 AI 好好说话”——这是所有后续学习的基础
  2. 再读 Claude Code 入门概览，建立 CC 基础认知
  3. 通读 CC “扩展能力层”（01-03）：理解 Skills → MCP → Tools 的扩展体系
  4. 学习 CC “协作执行层”（04/08/09/10）：从单 Agent 到多 Agent 编排
  5. 深入 CC “持久化与自动化层”（05-07）：Memory + Hooks + 配置管理
  6. 然后进入 Meta_Kim 专题：理解如何在 CC 基础之上叠加治理层
  7. 按 Meta_Kim 的 00 → 01 → … → 05 顺序阅读，理解从骨架到场景判断的完整治理体系
  8. 每个概念学习后，尝试在自己的项目中实践
  9. 回到总览，用 Obsidian 图谱视图（Graph View）发现概念之间的关联网络

💡 提示：在 Obsidian 中按 Cmd/Ctrl + E 切换编辑/预览模式，按 Cmd/Ctrl + K 插入内链。 🗺️ 目录树：每篇文章的”📂 目录树位置”章节展示了该概念在文件系统中的实际位置——所有概念最终都映射到 .claude/ 和 ~/.claude/ 目录下。
